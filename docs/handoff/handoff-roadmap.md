其中，HAMD、HAMA、照护负担等名称仅作为二期“更多认知、行为、情绪和日常生活能力量表”的候选示例，不构成逐量表承诺；绘图、连线和钟表任务的 AI 辅助观察仅作为二期 AI 证据辅助候选，不属于一期。独立 `FollowUpPlan` 模型、音频证据和面向业务用户的可视化量表配置后台当前不纳入一期或既定二期工作包，原始需求中的具体类名和早期模型命名也不构成实施约束；后续如需纳入，须由产品方重新明确确认并更新 roadmap，不得因历史提及自动恢复为隐含需求。

## 4. 二期规划

以下均为候选能力域，不是已承诺范围；进入二期前须完成业务、临床、科研、合规和资源确认。一期范围已纳入的基础趋势、基础科研导出、最小 AI 辅助、安全审计、独立 AuditLog、简易电子签名和报告作废闭环，不因下表中的增强规划而移出一期。

| 能力域 | 业务价值 | 一期预留底座 | 进入条件 | 关键风险 |
|---|---|---|---|---|
| 更多认知、行为、情绪和日常生活能力量表 | 扩展综合评估覆盖；ADL、IADL、CDR、FAQ、NPI、GDS、HAMD、HAMA、照护负担等仅为候选示例，不构成逐量表承诺 | 版本化量表配置、通用执行、作答、评分和认知域模型 | 获得量表授权、CRF 与评分规则确认，并完成逐量表临床验证 | 授权、版本差异、量表专有规则和误用风险 |
| 跨访视纵向分析增强 | 从一期基础趋势扩展到可比性校验、复查对比和纵向解释 | 访视、版本追溯、历史结果与一期趋势 | 明确随访周期、可比版本、缺失数据和解释规范 | 不同版本/施测条件不可比，趋势被误作诊断 |
| AI 辅助解释与医生审核增强 | 扩展多量表、趋势和证据辅助；绘图、连线和钟表任务的 AI 辅助观察仅为候选方向 | 一期受控 AI 草稿、报告审核、来源追溯和 LLM 配置预留 | 一期 AI 安全验收通过，模型、隐私、评测和降级方案稳定 | 幻觉、偏差、隐私泄露、自动化依赖和监管风险 |
| 科研数据集、脱敏与导出治理 | 支持可复用科研数据集、队列筛选、版本化字典和审批 | 一期基础导出、科研字段映射、证据链和权限审计 | 数据使用协议、伦理审批、脱敏标准和审批流程明确 | 重识别、越权使用、口径漂移和数据质量 |
| 多中心、机构与项目管理 | 支持机构、中心、科研项目和数据隔离 | 用户/角色、患者来源、配置和数据库治理底座 | 租户/机构边界、数据归属、跨中心协作与运维模型确认 | 数据串扰、权限复杂度、配置分叉和运维成本 |
| 辅助诊断规则与风险模型 | 在量表结果之外形成可解释的风险提示与临床决策支持 | 评分、认知域、趋势、报告与版本追溯 | 临床证据、验证数据集、阈值、责任边界和监管路径确认 | 将风险误作诊断、偏倚、不可解释和责任不清 |
| 患者端与家庭照护协同 | 支持提醒、材料补充、随访协作与照护沟通 | 患者/访视、会话、证据和消息配置预留 | 明确哪些任务可脱离医护监督、身份核验、隐私和可访问性要求 | 不得把核心量表变成未经监督的居家自测；数字鸿沟与隐私风险 |
| 高级权限、强身份签名与合规治理 | 支持细粒度且可配置权限、CA/PKI 数字签名、电子印章、可信时间戳、第三方电子签平台，以及长期合规归档与合规证明 | 一期基础用户/固定角色管理、独立 AuditLog、简易电子签名和报告不可变版本链 | 一期验收完成，并明确强身份认证、法规/院方、第三方认证及长期归档要求 | 权限复杂度与配置错误、签名法律效力和长期保留成本 |

## 5. 文档维护规则

1. roadmap 回答：现在完成到哪里、当前做什么、一期还剩什么、二期规划什么。
2. Git commits 回答：按时间顺序发生了什么。
3. backend/frontend snapshot 回答：当前代码真实实现了什么。
4. API / DTO / Service / Route / Component map 回答：实现在哪里、契约是什么。
5. decisions 回答：为什么这样设计。
6. testing playbook 回答：如何验证、实际验证到了什么程度。
7. roadmap 不再维护 commit 文件清单、每阶段测试数量、完整 API 明细、逐阶段执行结果或 A/B 流水账。
8. 阶段完成后，将结果压缩进“已完成能力”，整体替换“当前阶段”，更新“一期剩余工作包”，不得向 roadmap 追加长篇历史。
9. 一期范围变化时，必须记录新增、移除或调整的内容及原因，不得静默改变工作包总量。
10. 未确认事项必须明确标记“待确认”，不得包装为既定需求；候选二期能力不得写成合同承诺或当前实现。
