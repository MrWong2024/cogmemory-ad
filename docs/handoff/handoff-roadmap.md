# CogMemory AD / 智忆评 handoff 路线图

## 1. 文档定位

本文档用于记录 CogMemory AD / 智忆评 handoff 体系的路线图与维护口径，作为后续交接、复盘和阶段推进的索引型文档。

本文档只承接既有项目的 handoff 文档类型、治理方法和工程经验，不承接任何外部项目业务事实。

## 2. 当前承接状态

- 仓库已完成初始化。
- `docs` 下通用宪法文档已作为工程治理基线。
- 当前未发现 `backend\package.json`，后端工程尚未初始化，技术栈版本以后续 `package.json` 和实际代码为准。
- 当前未发现 `frontend\package.json`，前端工程尚未初始化，框架版本以后续 `package.json` 和实际代码为准。
- handoff 文档体系处于初始化阶段。
- 业务事实仍处于 MVP 范围确认阶段，具体量表、角色、流程、计分、报告和合规口径待后续业务文档确认。

## 3. 项目目标概述

CogMemory AD / 智忆评的项目方向是阿尔茨海默病认知评估与辅助诊断系统。

当前本文档不定义真实医疗诊断结论、量表计分规则、业务流程、角色权限、接口或数据模型。

## 4. 已完成能力

- 当前已确认：仓库与 `docs` 治理基线已就位。

2. 确认后端与前端初始化方案。
3. 建立最小可运行工程骨架。
4. 在业务文档明确后再设计接口、DTO、数据模型与页面路由。
5. 按实际实现同步 handoff 文档、验证手册与关键决策记录。
