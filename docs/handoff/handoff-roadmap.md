# CogMemory AD / 智忆评路线图

## 0. 项目总览

- 正式名称：智忆评——阿尔茨海默病认知评估与辅助诊断系统；CogMemory AD — Alzheimer’s Cognitive Assessment and Clinical Decision Support System。
- 产品定位：面向临床与科研场景的认知评估与辅助诊断系统，不是普通电子问卷，也不描述为患者居家自测产品。
- 一期核心范围：以 MMSE / MoCA 为核心，按可扩展产品底座建设量表配置与执行、作答和证据采集、评分复核、认知域、临床报告、历史随访、科研导出、AI 辅助、权限审计与安全验收。
- 长期基线：核心量表由医生、护士或研究人员陪伴或监督施测；图片、平板手写与可追溯证据链属于一期核心要求；重医已具备相关量表正规授权；明显 CRF 错误已按确认口径修正并保留版本追溯。
- 当前里程碑：后端完成至归档报告版本化更正（A25）；前端用户能力完成至版本化更正闭环（B15）。
- 当前业务阶段：replacement 后续生命周期（阶段编号启动前确认）。
- 一期已完成能力域：9 个。
- 一期剩余工作包：13 个，其中已确认 13 个、待确认 0 个。
- 已确认工作包规模：S 0 个、M 7 个、L 6 个；按 S=1、M=2、L=3 计，加权工作量为 `0×1 + 7×2 + 6×3 = 32`。
- 当前关键路径：WP-02 → WP-04 → WP-09。自动保存、科研导出、AI 与审计安全可在依赖满足后并行推进。
- 当前阻断项：无已知技术或范围确认阻断；范围已确认不等于接口、权限、状态和验收细节已全部设计完成，各工作包实施前仍须锁定具体契约和验收口径。
- 待确认事项：0 个。
- 下一阶段建议：replacement 后续生命周期（阶段编号启动前确认）；不预写未确认的 A/B 编号。

一期不按工作包数量计算或展示完成百分比。

## 1. 已完成能力

1. **工程、安全与认证底座**：NestJS、Next.js、MongoDB、Storage 抽象、配置校验、统一异常、服务端 Session + HttpOnly Cookie、角色 Guard 与安全响应边界已建立；当前只有认证与角色底座，没有公开用户管理和完整权限管理界面（A1–A11、B1）。
2. **患者与访视**：支持患者列表、创建、详情，访视列表、创建、详情，以及访视下量表实例初始化；患者/访视编辑、归档、删除、合并和完整状态流转尚未完成（A2、A12–A13，B2–B3）。
3. **MMSE / MoCA 量表配置与执行**：已建立版本化量表配置、种子校验、按需物化、执行实例与安全施测页面；不是配置管理后台，也尚无全量 seed runner（A1、A8–A9、A13–A14，B3–B4）。
4. **作答、计时、图片与平板手写证据**：支持逐题草稿、分步/提示后记录、计时字段、图片上传、平板手写画布、短期访问、作废重传与证据链；当前为逐题手工保存，尚无批量或自动保存调度和完整实时计时交互（A3–A4、A14–A16，B4–B6）。
5. **提交检查、评分与人工复核**：支持提交完整性检查、实例幂等提交、保守混合评分、待复核清单、单题人工评分、乐观并发与显式评分确认；评分确认不等于独立评分锁定（A5、A16–A18，B6–B8）。
6. **认知域计算与安全展示**：基于确认评分快照计算认知域结果，采用完整题分重叠归因并安全展示贡献与非诊断边界；尚无认知域人工确认、修改、重算或独立历史列表（A6、A19，B9）。
7. **临床报告生命周期**：后端已完成规则化非 AI 草稿、受控编辑、提交确认、医生/管理员确认、不可逆锁定、来源冻结、归档和线性版本化更正；前端 B15 已完成更正发起、恢复、幂等结果、source/replacement 追溯及合法 V2 的 A21 工作流接入。V2 锁定、冻结和归档尚未泛化（A7、A20–A25，B10–B15）。
8. **前端临床工作流**：已形成从认证、患者/访视、量表施测、证据采集、提交、评分复核、认知域到报告版本化更正的临床工作流；报告工作流 façade、coordinator、central state、单一写锁与单一 beforeunload 结构完成至 B15（B1–B15）。
9. **测试与质量保障**：已建立后端单元测试和隔离真实 HTTP E2E 基线，前端各阶段已执行静态检查与生产构建验证；真实浏览器联调、前端核心流程自动化与一期验收仍在剩余工作中。
