# CogMemory AD / 智忆评 前端验证手册

## 1. 文档定位

本文档用于记录 CogMemory AD 前端自动验证命令、人工验证口径、认证权限验证口径、医疗与隐私展示红线，供后续开发和交接使用。

## 2. 当前状态

- 当前前端根目录公共骨架配置已初始化。
- 当前 `frontend\app` 和 `frontend\src` 最小公共底座已初始化。
- 当前 `frontend\package.json` 已存在，自动验证命令以其中真实脚本为准。
- 当前不编写测试代码，不定义真实业务页面验证用例。
- 当前无真实登录、医生端、患者端、量表、评估记录、报告或 API client。

## 3. 自动验证命令

当前可见脚本包括：

- `npm run build`
- `npm run lint`
- `npm run typecheck`
- `npm run dev`
- `npm run start`
- `npm run lint:file`
- `npm run lint:file:fix`

本次任务执行状态：

- `npm install`：未执行。
- `npm run build`：未执行。
- `npm run lint`：未执行。
- `npm run typecheck`：未执行。
- 未执行原因：`frontend\node_modules` 不存在，按任务约束默认不执行安装和依赖于安装结果的构建 / lint / 类型检查。
- E2E：未执行；当前没有业务流程或 E2E 场景。

## 4. 当前应执行的非安装检查

本次公共底座任务完成时，应在仓库根目录执行：

- 旧标识 / 旧业务残留检查。
- 业务目录污染检查。
- `frontend\package.json` name 与端口检查。
- `git diff --check -- frontend docs\handoff`。
- `git status --short`。

检查结果以执行后的终端输出和任务最终汇总为准，不得编造通过结果。

## 5. 页面人工验证口径

当前首页人工检查项：

- 首页是否符合医疗系统 / 临床评估 / 低干扰 / 高可读性 / 冷静可信。
- 首页是否使用浅色背景、低饱和蓝绿、清晰分区、大字号可读、少动画、少装饰。
- 首页是否突出评估、记录、报告、医生工作流。
- 首页是否避免继承 ReviewX 视觉风格、颜色体系、页面布局、业务文案或管理后台气质。
- 首页是否无真实患者数据。
- 首页是否无真实医院数据。
- 首页是否无诊断结论暗示。
- 首页是否没有写成业务 MVP 已实现。

当前 not-found 人工检查项：

- 页面是否为 CogMemory AD / 智忆评口径。
- 页面是否提供返回首页入口。
- 页面是否不跳转业务路由。
- 页面是否不调用后端。

## 6. 认证与权限验证口径

- 认证与权限页面实现后，应补充未登录、已登录、权限不足、会话过期等状态验证。
- 当前认证与权限展示口径待后续业务文档和实际代码确认。
- 当前公共底座不实现认证状态和权限控制。

## 7. 量表作答与报告页面验证口径

- 量表作答、报告预览、医生端和患者端页面的验证细节待页面实现后补充。
- 当前不定义真实量表题目、计分规则、报告结论或页面流程。
- 后续验证量表作答页面时，应检查题目、选项、图片题和进度状态是否清楚，并确认页面没有无关装饰干扰。
- 后续验证报告页面时，应检查结果表达是否克制、准确、可解释，并确认系统结果没有被展示为最终医疗诊断。

## 8. 医疗与隐私展示红线

- 不展示真实患者敏感数据样本。
- 测试截图不得包含真实姓名、身份证号、手机号、病历号、住址等可识别信息。
- 量表图片题、报告预览、医生端和患者端页面的验证细节待页面实现后补充。
- 不得在页面文案或测试截图中呈现未经确认的真实医疗诊断结论。

## 9. 后续同步规则

- 前端新增或调整测试脚本后，应同步更新自动验证命令。
- 新增页面、路由、组件、API 对接或权限展示后，应同步补充对应验证口径。
- 新增页面、组件、布局、样式或关键交互后，应同步补充与 `handoff-frontend-design-baseline.md` 一致的视觉 / 可用性人工验证口径。
- 验证截图、测试数据和日志不得包含可识别个人信息。
