# CogMemory AD / 智忆评 前端验证手册

## 1. 文档定位

本文档用于记录 CogMemory AD 前端自动验证命令、人工验证口径、认证权限验证口径、医疗与隐私展示红线，供后续开发和交接使用。

## 2. 当前状态

- 当前前端根目录公共骨架配置已初始化。
- 当前 `frontend\package.json` 已存在，自动验证命令以其中真实脚本为准。
- 当前未执行 `npm install`。
- 当前未执行 `npm run build`。
- 当前未执行 `npm run lint`。
- 当前未执行 `npm run typecheck`。
- 未执行原因：本任务只迁移 `frontend\` 根目录配置，尚未迁移 `frontend\app` 或 `frontend\src` 页面入口，安装和构建类验证留待后续公共底座迁移后执行。
- 当前不编写测试代码，不定义真实页面验证用例。
- 当前已新增前端设计基线，后续页面人工验证需检查视觉与可用性是否符合该基线。

## 3. 自动验证命令

- 当前可见脚本包括 `dev`、`build`、`start`、`lint`、`lint:file`、`lint:file:fix`、`typecheck`，但本任务未执行安装、构建、lint 或类型检查。
- 后续迁移 `frontend\app` / `frontend\src` 公共底座后，再执行依赖安装、构建、lint 和类型检查。
- 后续只能记录真实存在的脚本，不得凭空假设未在 `frontend\package.json` 中定义的测试或 E2E 命令。
- 当前 `eslint.config.mjs` 保留公共 lint 口径中的 `react-hooks/exhaustive-deps` 与 `react-hooks/set-state-in-effect` 关闭项；这只记录当前公共配置，不代表业务代码可绕过代码评审。

## 4. 页面人工验证口径

- 页面实现后，应补充关键路径、加载态、空态、错误态和响应式展示检查项。
- 当前暂无已确认页面，具体人工验证步骤待页面实现后补充。

## 5. 视觉 / 可用性人工验证口径

后续前端页面验证时，应检查：

- 是否符合医疗系统 / 临床评估气质。
- 是否低干扰。
- 是否高可读性。
- 是否少动画。
- 是否少装饰。
- 是否突出评估、记录、报告、医生工作流。
- 是否避免继承 ReviewX 视觉风格、颜色体系、页面布局、业务文案或管理后台气质。
- 评估题目、患者信息、报告摘要和医生操作入口是否清楚可读。
- 医生端是否优先支持高效查看、发起评估、查看记录和生成报告。
- 患者端是否降低阅读负担和操作压力。

## 6. 认证与权限验证口径

- 认证与权限页面实现后，应补充未登录、已登录、权限不足、会话过期等状态验证。
- 当前认证与权限展示口径待后续业务文档和实际代码确认。

## 7. 量表作答与报告页面验证口径

- 量表作答、报告预览、医生端和患者端页面的验证细节待页面实现后补充。
- 当前不定义真实量表题目、计分规则、报告结论或页面流程。
- 后续验证量表作答页面时，应检查题目、选项、图片题和进度状态是否清楚，并确认页面没有无关装饰干扰。
- 后续验证报告页面时，应检查结果表达是否克制、准确、可解释，并确认系统结果没有被展示为最终医疗诊断。

## 8. 医疗与隐私展示红线

- 不展示真实患者敏感数据样本。
- 测试截图不得包含真实姓名、身份证号、手机号、病历号、住址等可识别信息。
- 量表图片题、报告预览、医生端和患者端页面的验证细节待页面实现后补充。
- 不得在页面文案或测试截图中呈现未经确认的真实医疗诊断结论。

## 9. 后续同步规则

- 前端新增或调整测试脚本后，应同步更新自动验证命令。
- 新增页面、路由、组件、API 对接或权限展示后，应同步补充对应验证口径。
- 新增页面、组件、布局、样式或关键交互后，应同步补充与 `handoff-frontend-design-baseline.md` 一致的视觉 / 可用性人工验证口径。
- 验证截图、测试数据和日志不得包含可识别个人信息。
