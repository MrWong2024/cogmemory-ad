# CogMemory AD / 智忆评 前端路由地图

## 1. 文档定位

本文档用于记录 CogMemory AD 前端路由、页面职责、访问边界和数据来源，供后续开发、测试和交接使用。

## 2. 当前状态

- 当前只初始化 app/src 公共底座路由。
- 当前不包含真实登录、医生端、患者端、量表、评估记录或报告路由。
- 当前不包含任何业务工作台。
- 当前不包含页面级后端 API 调用。

## 3. 当前路由清单

- 路径：`/`
- 页面名称：公共首页占位
- 页面职责：展示 CogMemory AD / 智忆评公共底座状态，标识评估、记录、报告、医生工作流等后续方向
- 访问角色：公开
- 数据来源：静态内容，无 API 调用
- 加载态：无异步加载
- 空态：不适用
- 错误态：不适用
- 关联组件：`Badge`、`Card`
- 备注：非业务工作台，后续可被真实首页或登录流程替换 / 扩展；当前不代表业务 MVP 已实现

- 路径：`not-found`
- 页面名称：404 兜底页
- 页面职责：处理未匹配地址，提供返回首页入口
- 访问角色：公开
- 数据来源：静态内容
- 加载态：无异步加载
- 空态：不适用
- 错误态：页面自身即为兜底展示
- 关联组件：`Badge`、`Card`
- 备注：不跳转业务路由，不调用后端

## 4. 后续同步规则

- 路由事实以实际前端代码、路由文件和业务文档为准。
- 不得在页面未实现前写成已存在路由。
- 路由涉及接口调用时，应同步更新 `docs\handoff\handoff-frontend-api-map.md`。
